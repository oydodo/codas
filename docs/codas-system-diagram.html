Codas 系统图 · 从 code → fact → claim 逐层构建

Codas = Code Atlas System。核心思想：把一个仓库的“真相”分成三类，再用策略闸门强制三者一致。下面 6 步累加，每步在上一步基础上加一层。数字来自当前仓库实时图谱（codas schema）。

1

基石三类真相

为什么要分三类？代码是真相但是“哑”的。有些东西能从代码机械推导（谁调用谁 = 事实）；有些推导不出来——“这两个重复函数是故意的”“X 改了 Y 必须跟着改”——这只能由人来断言 = 主张。机器证不了的，人补上。

CLAIM 主张
人写意图 · 事实证不了的关系/义务
一句话：CODE 是“有什么”，FACT 是“能推导出什么”，CLAIM 是“人断言了什么”。整个 Codas 就建在这三者的区分上。

两层正确性 — GATE = 机器能证的硬不变式（依赖方向、重复、漂移）；INJECTION = 机器证不了的承重项（放对子系统、语义复用、抽象层级）→ 在 agent 写之前喂给它。闸门兜底，注入预防。
