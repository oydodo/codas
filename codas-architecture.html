3 · 代码事实层(open-world)现状

层2 拆 gate-grade(确定·语言内·拦) 和 advisory(多语言+跨语言·提示·不拦)。三个 adapter 槽:

adapter 槽填谁tier用途
parse(层1)ast(Python)/ tree-sitter(每语言)—源码→语法树
symbols / importstree-sitter / ast(语言内,确定)gateownership / dup / dependency-direction 拦
call-graph(语言内)ast(Python)· tree-sitter resolver(某语言要 call-gating 才搓,少见)gatefact_coupling on calls(少)
call-graph(多语言+跨语言)CodeGraph adapter(可插拔)advisoryimpact / reuse / 跨语言理解 / code_anchor 多语言

ScanContext = 事实提供 seam(单次扫描,policy 与 inventory 共用)。跨语言 = 必须支持(advisory),但猜的边不进 gate(防假拦);gate 只烧语言内确定事实。

open-world 不变量:静态读能证"存在",不能证"不存在"(动态/反射看不见)→ 事实是下界,absence≠denial。gate 只能基于"确定存在"拦。

源头切分(消歧关键):代码 → 事实(facts)(symbols/imports/calls,不叫 claim);文档/配置 → claim。代码从不产 claim。"从代码发现的 claim"其实是从文档扫出的 doc claim。

4 · Atlas / inventory(范围 + hash)现状

事实+结构+claim+任务的确定 JSON 投影。被 hash 的就是这份。

IN — 三组(都进 inventory + 进 hash)

A. authored 配置(closed)
units / unowned / conflicts ← structure.yml · program.work_items · documents.roles
B. claims(断言)
doc_claims · html_claims · wiki_claims(verified 锚点)
C. 事实(open)+ 任务
symbols · imports · calls(代码,ast)· tasks(Trellis)

OUT — 不在 inventory(也不在 hash)

prose
.codas/wiki/code/*.md 散文 · LLM 离线写
渲染的书 wiki/
派生输出 · scanner 排除
机器 scratch
.install-state / .status-seen
working-tree delta
changed_paths/snapshots · 不序列化

三个 hash

hash算什么范围
inventory_hash(主)sha256(整份 inventory JSON,sort_keys)A+B+C 整块 ← 过宽
generated 页 source_hash只 units+roadmap 子集已 scoped(先例)
policy_versionpolicies 配置独立
⚠️ 主 hash 盖 A+B+C 整块 → 改一行代码(C)→ 连 A 的 structure 文档、B 的 doc_claims 都判"可能过期"(churn)。prose / 书 / scratch / delta 在范围外 → prose 变靠 diff 抓,不靠 hash(hash 只是 facts/inventory 的指纹,漏 prose+渲染器)。提案:退整份 hash → 统一 diff(re-render + byte-compare),留渲染确定性测试。

消费者:codas inventory · query · provenance(hash→receipt)· wiki/book 渲染 · preflight。gate 不读它(用内存事实)。fact 格式 = 扁平记录,如 {module,name,kind,line}。

6 · 结构图 & 归属现状

每个被治理路径 → 一个单元(unit)→ 一个 owner + canonical placement。

unit
id · path · kind · owner · status · canonical_placement · must_not_depend_on
gate 能查
改文件是否被某单元拥有(missing_owner)· 是否对齐(structure_drift)
gate 查不了(ungateable)
放错子系统(owned 但错)· 语义重复 · 错抽象层

7 · claim / 锚点层(代码↔文档桥)现状

"claim" = 一个属:关于代码的、可被事实核对的断言。全都拿同一套事实去核。下面是三个种(别裸叫 claim,带限定词):

种哪谁产断言什么enforcement
关系/治理 claimclaims.yml(§2)authored 主动写关系/意图(重复有意 · 须连改)closed · 拦
doc claimmarkdown 文档extracted 扫出文档引用了某路径/符号核对没说谎
锚点 claim.codas/wiki/code/**authored 写在页里散文绑符号 · A calls Bopen warning

Codas 读不了散文意思(无 LLM,§17)→ 只能核对带"锚点/引用"的断言。锚点解析不上 = 定位到代码↔文档差异。下面是实现这些的 policy:

机制(policy)方向语法/来源类型
code_anchordoc → 代码事实defines: / calls: / contains: 在 .codas/wiki/code/**advisory
stale_claimdoc → 路径markdown 链接/code-span 路径advisory
fact_coupling代码变 → doc 须同改.codas/claims.yml 手写耦合gated
问题3(决策没回写 PRD)的引擎就在这层。缺口:PRD/决策文档还没接锚点(提案:扩锚点范围 + 从锚点派生耦合 + RepairTarget)。
"claim 重载"的根治(不是改名,是合并):claims.yml 的 fact_coupling(治理 claim)和文档锚点是同一件事两种写法。提案:写一条锚点,Codas 既核"文档没说谎"(doc 侧)又派生"连改强制"(治理侧)→ 一个 source 两个作用。合了之后"两种 claim"的混淆从根上消失。

14 · 双层正确性模型(核心理念)现状

(a) 硬不变量 → GATE 保证
missing_owner · structure_drift · deprecated_path · duplicate_implementation · dependency_direction · byte-identical。check+verify+inventory 兜。
(b) ungateable → 注入防御
放错单元(owned 但错)· 语义 reuse miss · 错抽象层。gate 沉默 → 注入 ownership/reuse/影响面是唯一防线 = 对正确性 load-bearing。

15 · 未来方向(提案,5 顾问裁决)提案

① 问题3 锚点(最值钱,零 CodeGraph)
决策/PRD 文档接锚点 + 从锚点派生耦合 + RepairTarget(精确指出哪行过期+改成啥)。层3 + Python 层2,机制+数据现成。

整份 hash → 每 committed 物 scoped hash(structure/code/book 各一)。不退役,只收窄。

post-merge/post-rewrite(今天就有的洞,独立于 CodeGraph)。
被否:大改(全走 CodeGraph + diff-gate + 退役 inventory)—— 回归确定/防绕过/离线,且 diff-gate 在 merge/rebase 漏判。CodeGraph 永远到不了层3。
