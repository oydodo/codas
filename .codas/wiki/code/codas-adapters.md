The adapters package is Codas's **ingestion boundary** — the only code allowed to touch the messy outside world (the filesystem, git, raw `.py`/`.md`/`.html`/`task.json` bytes) and turn it into frozen, sorted, deterministic fact dataclasses. Everything downstream (policies, the app layer that assembles `ScanContext`) consumes those facts and never imports an adapter, so the raw-IO surface stays small, auditable, and replaceable. Each extractor is a pure function of *(file-set, content)*: it reads, parses with stdlib only (`ast`, `html.parser`, `re`, `json`), sorts by a total key, and de-dups — no model, no per-process nondeterminism, so the inventory hashes byte-identically across runs and machines.

The fact families split cleanly. `python_parse.parse_python_modules` does the single shared `ast.parse` pass; `python.extract_symbol_facts_from_parsed` and `extract_import_facts_from_parsed` project symbols and import edges from it, and `callgraph.extract_call_facts_from_parsed` resolves first-party call edges, tagged by how they resolved (`direct`, `imported_symbol`, `module_attribute`, `self_method`). The doc family — `markdown.extract_doc_claims`, `html.extract_html_claims`, `wiki.extract_wiki_claims`, `semantic.extract_semantic_claims` — extracts path/structural claims from prose; the HTML and wiki adapters deliberately *reuse* markdown's `_normalize`/`_resolve` path-shape gate rather than fork it (the `duplicate_implementation` trap). `git.py` reads working-tree drift and `HEAD` blobs for the coupling and snapshot paths.

Two invariants run through all of it. First, **open-world soundness**: the call graph emits only edges that resolve to a real first-party definition and drops dynamic/builtin/unresolved calls rather than guessing — the output is a sound lower bound, never a closed-world denial. Second, **package membership is derived from the file SET, not the live filesystem** (`package_dirs_of`, `dotted_for`), so a snapshot taken at any git ref computes identically regardless of the dirty working tree. `ParsedModule`'s three-state read/parse/error model lets each extractor reproduce its exact historical skip-vs-propagate policy from one shared read, and advisory prose (e.g. `.codas/wiki/code/`) is kept out of the hash — only its structural claims are verified.
