How the gate treats each policy: **gated** = an error it blocks on; **advisory** = a
warning it surfaces; **planned** = declared but not yet enforced.

## Ungateable failure classes (the gate is SILENT here)

The gate formalizes only a subset of correctness. These design-quality failures are
caught by NO active policy today — the ownership map below and a reuse query are your
only defense, so verify them yourself before writing:

- Wrong unit but still owned — code placed in an owned path that is the wrong subsystem (missing_structure_owner only checks that SOME unit owns it, not that it is the RIGHT one).
- Semantic reuse miss — re-implementing a concept that already exists (duplicate_concept is planned; duplicate_symbol only flags exact top-level name clashes).
- Wrong abstraction or hook point — a change that type-checks and is owned but sits at the wrong layer or seam.

## Lookup protocol

The live atlas is fresh and deterministic. Query it (the rendered `wiki/` book is for
humans; these commands are the agent's table):

- `codas preflight --task <id>` — read-first context, reuse candidates and affected units.
- `codas query <kind>` — one fact block (symbols, imports, calls, units, …); add
  `--select FIELD=VALUE` to filter.
- `codas impact <symbol-or-path>` — what transitively calls a symbol or file.
- `codas schema` — the queryable fact and field shape.
- `codas check .` — the gate; run it before you commit.
